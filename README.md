# Voronoi‑ICOBI Coordination Analyzer

**Robust coordination number analysis from Voronoi polyhedra with optional bonding validation using LOBSTER ICOBI values.**

---

## Overview

This script computes coordination environments in periodic structures using a **Voronoi tessellation** approach applied to a VASP `POSCAR` file. It optionally integrates **ICOBI (Integrated Crystal Orbital Bond Index)** values from **LOBSTER (tested with version 5.1.1)** to distinguish true bonding interactions from purely geometrical neighbors.

The code is designed for:

- Solid‑state chemistry
- Materials science
- Bonding analysis
- Coordination number determination
- Automated structural analysis workflows

It supports:

- Strict Voronoi coordination number (geometric)
- Effective coordination number (ECoN)
- Face‑area filtering
- ICOBI‑based bonding filtering
- Full periodic boundary conditions
- Multi‑atom batch analysis

---

## Features

- Pure Python implementation
- No dependence on VASP runtime
- Works with any periodic crystal structure
- Robust handling of periodic images
- Optional chemical bonding validation using ICOBI

```
--atom INT
```

Index of central atom (0‑based).

```
--all
```

Compute coordination for all atoms.

```
--cutoff FLOAT
```

Periodic image search radius (Å).

Default:

```

```

Recommended range:

```

```

### Coordination Metrics

```
--econ
```

Compute effective coordination number (ECoN).

Two definitions are reported:

- distance‑weighted ECoN
- face‑area‑weighted ECoN

---

### Voronoi Filtering

```
--threshold FLOAT
```

Minimum fraction of total Voronoi face area required to keep a neighbor.

Example:

```
--threshold 0.02
```

Meaning:

```
Exclude faces smaller than 2% of total surface area
```

Typical values:

```
0.01 – 0.05
```

---

### ICOBI Bonding Options

```
--icobilist FILE
```

Path to:

```
ICOBILIST.lobster
```

Example:

```bash
--icobilist ICOBILIST.lobster
```

---

```
--icobi-threshold FLOAT
```

Minimum ICOBI value to consider a neighbor as bonded.

Example:

```
--icobi-threshold 0.01
```

Typical interpretation:

```
ICOBI ≥ 0.5   strong bond
ICOBI ≥ 0.1   moderate bond
ICOBI ≥ 0.02  weak bond
ICOBI < 0.02  negligible
```

---

```
--show-excluded
```

Display neighbors removed by filtering.

## Scientific Methodology and Interpretation

### Purpose of the Method

The goal of this tool is to determine **chemically meaningful coordination environments** in crystalline materials by combining:

1. **Voronoï tessellation** (geometric neighbor detection)
2. **Face-area filtering** (topological significance)
3. **ICOBI filtering from LOBSTER** (chemical bonding validation)
4. **Effective coordination number (ECoN)** (continuous coordination measure)

This multi-step approach resolves a well-known limitation in coordination analysis:

```
Geometric coordination ≠ Chemical coordination
```

Purely geometric methods often overestimate coordination numbers, especially in:

- ionic compounds
- hydrides
- high-symmetry metals
- open frameworks
- distorted environments

The present method provides a reproducible and physically interpretable alternative.

---

### Coordination Definitions Used

The script reports three coordination numbers, each corresponding to a different physical meaning.

#### 1) Raw Voronoï Coordination Number

```
CN_raw
```

Definition:

```
Number of atoms sharing a Voronoï face
```

Properties:

- purely geometric
- sensitive to small faces
- typically overestimates bonding

Use case:

```
Topological analysis
```

---

#### 2) Area-Filtered Coordination Number

```
CN_filtered
```

Definition:

Neighbors are retained only if their Voronoï face area exceeds a fraction of the total surface area.

Filtering rule:

```
face_area_fraction ≥ threshold
```

Typical threshold:

```
0.01 – 0.05
```

Use case:

```
Structural coordination
```

---

#### 3) ICOBI-Filtered Coordination Number

```
CN_chemical
```

Definition:

Neighbors are retained only if the bond strength exceeds a minimum value.

Filtering rule:

```
ICOBI ≥ threshold
```

Recommended default:

```
ICOBI ≥ 0.02
```

Physical meaning:

```
Chemically bonded neighbors
```

Use case:

```
Chemical coordination number
```

---

### Recommended Default Parameters

These values have been validated on ionic and covalent systems, including ternary hydrides.

```

ICOBI threshold    = 0.02

```

These parameters provide a robust balance between:

- numerical stability
- physical realism
- reproducibility

---

### Effective Coordination Number (ECoN)

The script implements the definition proposed by O'Keeffe.

Interpretation:

```
ECoN ≤ CN_filtered
```

ECoN measures the effective number of neighbors after distance weighting.

Typical reference values:

```
Tetrahedral      ECoN ≈ 4
Octahedral       ECoN ≈ 6
Cubic            ECoN ≈ 8
Close-packed     ECoN ≈ 12
```

ECoN is particularly useful for:

- distorted polyhedra
- mixed bonding environments
- quantitative comparison between structures

---

### Example Interpretation Workflow

Recommended analysis sequence:

```
Step 1   Compute raw Voronoï neighbors
Step 2   Remove negligible faces
Step 3   Remove non-bonding contacts
Step 4   Evaluate ECoN
```

This produces a coordination number that is:

```
physically meaningful
chemically interpretable
reproducible
```

---

### Example Scientific Behavior

Typical observation in ionic hydrides:

```
CN_raw        = 13
CN_filtered   = 7
CN_chemical   = 7
ECoN          ≈ 4
```

Interpretation:

- Voronoï geometry detects many weak contacts
- Area filtering removes negligible neighbors
- ICOBI filtering confirms chemical bonding
- ECoN quantifies bond strength distribution

This behavior is consistent with known coordination environments in ionic and mixed-bonding systems.

Compared to traditional coordination analysis, this approach provides:

- integration of electronic structure information

### Limitations

Users should be aware of the following constraints:

- Requires reliable electronic structure calculation for ICOBI filtering
- Very small thresholds may introduce numerical noise
- Extremely open structures may require larger cutoff radii
